웹 성능 최적화는 사용자 경험에 직접적인 영향을 미치는 중요한 요소입니다. 빠른 로딩 속도는 이탈률을 줄이고 전환율을 높입니다.

## 이미지 최적화

1. **적절한 포맷 선택**: WebP, AVIF 등 최신 포맷 활용
2. **Lazy Loading**: 필요할 때만 이미지 로드
3. **반응형 이미지**: `srcset`으로 다양한 크기 제공

```html
<img 
  src="image.jpg" 
  srcset="image-small.jpg 480w, image-large.jpg 1200w"
  loading="lazy"
  alt="설명"
/>
```

## 코드 분할 (Code Splitting)

```javascript
// 동적 import로 코드 분할
const Component = lazy(() => import('./Component'))

// React에서 사용
<Suspense fallback={<Loading />}>
  <Component />
</Suspense>
```

## 번들 최적화

- **Tree Shaking**: 사용하지 않는 코드 제거
- **Minification**: 코드 압축
- **Gzip/Brotli 압축**: 전송 데이터 크기 감소

## 캐싱 전략

- **브라우저 캐싱**: 정적 자산에 적절한 캐시 헤더 설정
- **CDN 활용**: 지리적으로 가까운 서버에서 콘텐츠 제공
- **Service Worker**: 오프라인 지원 및 캐싱

성능 최적화는 지속적인 모니터링과 개선이 필요한 과정입니다.
